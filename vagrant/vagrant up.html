vagrant up

This command will finish and you'll have a virtual machine running When you're ready to come back to your project, whether it is tomorrow, a week from now, or a year from now, getting it up and running is easy: $ vagrant up That's it! Since the Vagrant environment is already all configured via the Vagrantfile, you or any of your coworkers simply have to run a vagrant up at any time and Vagrant will recreate your work environment. Providers Vagrant can work with a wide variety of backend providers, such as VirtualBox, VMware, AWS, and more. Read the page of each provider for more information on how to set them up. Once you have a provider installed, you don't need to make any modifications to your Vagrantfile, just vagrant up with the proper provider and Vagrant will do the rest: $ vagrant up --provider=vmware_fusion For more information on providers, read the full documentation on providers. (https://docs.vagrantup.com/v2/providers/)
